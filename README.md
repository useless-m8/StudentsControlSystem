# Информационная система учета успеваемости студентов

Проект реализован по стеку из курсовой работы:

- клиентская часть: React + TypeScript + Vite;
- серверная часть: TypeScript + Express.js;
- база данных: PostgreSQL;
- обмен данными: REST API.

## Возможности

- авторизация пользователей `admin / admin` и `user / user`;
- регистрация новых студентов с указанием ФИО, года поступления, формы обучения и группы;
- авторизация через JWT-токены;
- разграничение прав: студент просматривает свою запись, свои оценки и данные своей группы, администратор видит все данные и управляет записями;
- учет студентов, групп, форм обучения, специальностей и дисциплин;
- ведение учебного плана;
- ведение журнала успеваемости;
- отчет по студентам, среднему баллу и задолженностям;
- визуальное выделение неудовлетворительных оценок и незачетов.

## Запуск

1. Установить зависимости:

```bash
npm install
```

2. Запустить PostgreSQL через Docker:

```bash
docker compose up -d
```

3. При необходимости скопировать `.env.example` в `.env` и изменить параметры подключения к БД.

4. Запустить клиент и сервер:

```bash
npm run dev
```

Клиент будет доступен на `http://localhost:5173`, API сервер на `http://localhost:4000`.

При первом запуске сервер автоматически создает таблицы PostgreSQL и добавляет демонстрационные данные.

## Проверка сборки

```bash
npm run build
```

Команда проверяет TypeScript для клиента и сервера, затем собирает фронтенд.

## Тестирование

Запуск тестов:

```bash
npm test
```

В проекте используются unit-тесты на Vitest. Сейчас набор содержит 18 тестов, разделенных по иерархии:

- `tests/server` - тесты серверной логики;
- `tests/client` - тесты клиентской бизнес-логики.

Что проверяется:

- JWT-авторизация: создание токена, чтение данных пользователя из токена, сохранение роли, группы и студента, отклонение неверной подписи и поврежденного токена;
- разграничение доступа: администратор видит все данные, студент ограничивается своей группой и своей записью;
- отчеты по успеваемости: фильтрация студентов по группе, расчет среднего балла, количества оценок, задолженностей, обработка студентов без оценок.

Файлы тестов:

- `tests/server/auth.test.ts`;
- `tests/server/accessScope.test.ts`;
- `tests/client/reportUtils.test.ts`.

## Основные API маршруты

- `POST /api/auth/login` - вход в систему;
- `GET /api/app-data` - получение всех данных приложения;
- `POST /api/students`, `PUT /api/students/:id`, `DELETE /api/students/:id` - студенты;
- `POST /api/groups`, `PUT /api/groups/:id`, `DELETE /api/groups/:id` - группы;
- `POST /api/directories/:kind`, `PUT /api/directories/:kind/:id`, `DELETE /api/directories/:kind/:id` - справочники;
- `POST /api/curriculum`, `PUT /api/curriculum/:id`, `DELETE /api/curriculum/:id` - учебный план;
- `POST /api/performance`, `PUT /api/performance/:id`, `DELETE /api/performance/:id` - журнал успеваемости;
- `GET /api/reports/summary` - сводные показатели.
